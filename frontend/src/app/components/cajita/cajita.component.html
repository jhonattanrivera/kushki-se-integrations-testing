Cajita

El uso de Cajita como librería de FrontEnd asegura que la información sensible de los pagadores, específicamente los datos de Tarjetas de Crédito, sea tratada, procesada, y transmitida de forma segura sin que esta información pase por los servidores del Comercio; reduciendo así el Scope PCI del Comercio.

Nota: El diseño de los formularios generados por Cajita, para cada medio de pago, es predefinido por Kushki y no puede ser modificado ni en estructura, ni en contenido.

Implementación de la Librería

  Pagos Únicos   WebPay   Recurrencias
Cajita para Pagos Únicos presenta los campos necesarios para que el Comercio obtenga el Token requerido para complementar el flujo de generación del cobro Cajita para Pago Único con WebPay presenta los campos necesarios para que el Comercio obtegna el Token requerido para complemntar el flujo de pago y generar el enlace correspondiente Cajita para Suscripciones Calendarizadas presenta los campos necesarios para que el Comercio obtenga el Token requerido para complementar el flujo de creación de Suscripción
